# 1. シーンの基本

PLCKでは、 `video` / `slide` / `quiz` / `html` / `chat` の5つのシーンを使って講座コンテンツを作成することができます。

ここでは、各シーンを構成するファイルなどの基本的な内容を解説します。

1. シーンを構成するファイル
    1. config.*yaml
    1. customHook.js
    1. style.css
    1. html.vue
1. シーンのディレクトリ構造
1. 画像などのアセットファイルの読み込みについて


## 1. シーンを追加する

シーンを追加する場合は、PLCKのコマンドラインを用いて作成することができます。

### `npm run add <シーンタイプ> <シーン名> <ユニット名>`

`npm run add <シーンタイプ> <シーン名> <ユニット名>`　を実行することで、シーンを追加できます。<br />

#### シーンタイプについて

シーンタイプには、 作成したいシーンのタイプである `video` / `slide` / `quiz` / `html` / `chat` のいずれかを指定します。<br />
もし、チャットタイプのシーンを作成したい場合は、以下のようなコマンドラインを実行します。

```shell
$ npm run add video test-video
```

#### シーン名について

シーン名は任意のシーンの名前を指定します。指定できるのは半角英数字と　-（ハイフン） ・ _ (アンダースコア)となります。<br />
指定したシーン名はそのままそのシーンのフォルダの名前となります。

#### ユニット名について

シーンを作成する際に、ユニット名を指定することができます。<br />
ユニット名を指定すると、そのユニットの言語設定に合わせた `config.*.yaml` ファイルを生成します。<br />
ユニット名は、 `units/` ディレクトリで指定されている各ユニットのフォルダ名です。<br />
もし、 `units/sample-unit` の設定を作成するシーンに設定したい場合は、

```shell
$ npm run add video test-video sample-unit
```

となります。

ユニット名を省略した場合、ユニットの一覧の一番上のユニットの設定が適用されます。

**注意:**<br />
ユニットがまだ作成されていない場合は、シーンを作成がエラーとなります。<br />
先にユニットを作成してから、シーンの作成を行ってください。<br />
ユニットに関しては[こちらをご参照](../unit.md)ください。


## 1. シーンを構成するファイル

シーンは以下のファイルで構成されています。

| ファイル名 | 概要 | ファイルが生成されるシーン |
|------|----|----|
| config.*.yaml | シーンの設定を記述する基本となるファイルです。 * は ja や en などの言語のロケール文字列となり、設定されている各言語ごとの設定が必要です。 | 全てのシーン |
| customHook.js | シーンに設定したい追加のスクリプトを記述することができます。シーンごとに設定されているイベントに応じて処理を記述することができます | 全てのシーン |
| style.css   | シーンのスタイルを変更することができます。作成したばかりの時は、デフォルトのスタイルが設定されています。 | video / quiz / slide |
| html.vue | シーンのコンテンツのHTMLを記述することができます。 | html / chat |


### 1. `config.*.yaml`

各シーンの設定が書かれているファイルです。<br />
シーンごとに設定できる内容が違います。<br />

この設定ファイルは言語ごとにファイルが作成され、 `*` には、言語のロケール文字列が入ります。たとえば日本語（`ja`）と英語（`en`）が設定されている場合は、 `config.ja.yaml` と `config.en.yaml` の2ファイルが生成されます。

生成される言語ごとのconfigファイルは、`plck.config.yaml` で設定される以下の項目に応じて作成されます。

```yaml
multi_lang: on
default_lang: ja
lang:
  - ja
  - en
```

| プロパティー名 | 概要 | 値 | 入力例 |
|------|----|----|----|
| multi_lang | 複数の言語の有効の有無を指定します。<br />有効となっている場合は、画面上に言語を選択するセレクトボックスが表示されます。 | `on` もしくは `off` | `off` |
| default_lang | デフォルトで表示する言語を指定します | [言語コード](./locale.md) | `ja` |
| lang | この講座で使用できる言語を指定します。 | [言語コード](./locale.md)の配列 | こちらを参照 |


また、`npm run add` コマンドは、新しいシーンを生成する際に設定されている言語ごとのファイルを自動的に生成します。


### 2. `customHook.js`

`customHook.js` は、各シーンごとに設定されているイベントが発生した時に実行できる外部スクリプトを定義することができます。<br />
たとえば、 `video` シーンの場合、 `videoInitialized` イベントが用意されており、ビデオが初期化されたら実行できるスクリプトを埋め込み処理をカスタマイズすることができます。

**customHook.js**
```js
export function videoInitialized() {
    console.log('test')
    // ここにビデオの初期化が完了した後に実行するスクリプトを実行する
}
```

#### JavaScriptの記述方法について

PLCKは、内部的に `Vue3` という `React` と並ぶ人気を誇るフロントエンドのフレームワークを使用しています。<br />
そのため、標準的に `import` などの `ES6` 構文を使用することが可能です。<br />
もし これまで`Babel` などを用いて `ES6` を使用した経験があれば、それらを活用することができます。

```js
// npm でパッケージとしてインストールしたjqueryを使用する
import jQuery from 'jquery'

const $ = jQuery
export function videoInitialized() {
    // jQueryを使ってDOMを操作する etc..
    $('.video').css("display", "none")
}
```

#### フレームイベントについて

他にも、ユニットが完了状態になった時や、特定のシーンが完了状態になった時などに実行されるフレームイベントというものがあります。<br />
フレームイベントについては、[フレームについて](../frame.md)で解説されています。


### 3. `style.css`

`style.css`には、シーンに関連するスタイルが記述されています。<br />
生成された当初は、デフォルトのスタイルが記述されていますので、適宜スタイルを変更するなどして見た目を調整することができます。

#### フレームスタイルについて

メニューやヘッダーなどのシーンフレーム全体におよぶスタイルをフレームスタイルを用いて変更することができます。
フレームスタイルについては、[フレームについて](../frame.md)で解説されています。


### 4. `html.vue`

`html`シーンと`chat`シーンでは、任意のHTMLを自由に記述することが可能です。
`.vue`となっているように、`vue`のテンプレートとほぼ同様の機能を使用することができます。

**注意**: `html.vue`では HTMLの部分となる `template` と、スタイルとなる `style` のみが変更可能です。`script` へのアクセスは基本制限されています。

```html
<template>
    <p>この中にHTMLを記述します</p>
</template>
<style>
p {
    color: red;
}
</style>
```



## 2. シーンのディレクトリ構造

各シーンにはそれぞれディレクトリ構造を持っており、ファイルの読み込みの優先順位が決められています。<br />
先ほど紹介したファイルのうち、`style.css` と `customHook.js` は、`npm run add`でのファイル生成時、各シーンの種類ごとフォルダの直下に作成されます。

<pre>
┣ scenes
┃   ┣ html
┃   ┃  ┣ html-scene1
┃   ┃  ┃     ┣ config.*.yaml 
┃   ┃  ┃     ┣ html.vue 
┃   ┃  ┣ html-scene2
┃   ┃  ┃     ┣ config.*.yaml 
┃   ┃  ┃     ┣ html.vue 
┃   ┃  ┣ style.css 
┃   ┃  ┣ customHook.js 
...
</pre>

例えば、`html`シーンの場合、上記の例のように `html-scene1`などの各シーン名の配下に `config.*.yaml` と `html.vue` が設置され、 `style.css` と `customHook.js` は シーンの種類フォルダの`html`配下に設置されています。

この状態だと、`html/style.css` と `html/customHook.js` が `html-scene1` と `html-scene2` それぞれに適用され、同じ見た目、同じ外部スクリプトを実行することになります。

もし、`html-scene1` のみに適用したい `style.css` と `customHook.js` があったとします。その場合は、 `html/html-scene1` 配下に、同じように `style.css` と `customHook.js` の両方を配置します。


<pre>
┣ scenes
┃   ┣ html
┃   ┃  ┣ html-scene1
┃   ┃  ┃     ┣ config.*.yaml 
┃   ┃  ┃     ┣ html.vue 
┃   ┃  ┃     ┣ style.css # style.cssと...
┃   ┃  ┃     ┣ customHook.js # customHook.js を設置する
┃   ┃  ┣ html-scene2
┃   ┃  ┃     ┣ config.*.yaml 
┃   ┃  ┃     ┣ html.vue 
┃   ┃  ┣ style.css 
┃   ┃  ┣ customHook.js 
...
</pre>

こうすることで、 `html-scene1` には 自身の配下にある `style.css` および `customHook.js` の適用が優先されます。<br />
この時、 `html-scene1` には `html/style.css` と `html/customHook.js` は適用されないことになりますので注意しください。

このようにして、個別の見た目や処理内容をシーンごとに設定することも可能です。


## 3. 画像などのアセットファイルの読み込みについて
 
シーンで使用する画像などの外部ファイルの読み込みについては以下の通りです。

### `scenes`ディレクトリにファイルを設置する

`scenes`ディレクトリに設置したフォルダやファイルを参照することができます。

たとえば、 `scenes/html/html-scene1` ディレクトリに `img` フォルダを用意し、 `demo.png`ファイルを設置した場合、`html.vue`で以下のように記述できます。

**フォルダ構造**
<pre>
┣ scenes
┃   ┣ html
┃   ┃  ┣ html-scene1
┃   ┃  ┃     ┣ config.*.yaml 
┃   ┃  ┃     ┣ html.vue 
┃   ┃  ┃     ┣ img 
┃   ┃  ┃     ┃  ┣ demo.png # 画像ファイル
...
</pre>

**html.vue**
```html:html.vue
<template>
    <div>
        <!-- demo.pngを読み込む -->
        <img src="@scenes/html/html-scene1/img/demo.png" />
    </div>
</template>
```

画像のパスを指定する時に、 `@scenes/~` と書くことでことで、 `scenes/~` 配下を参照することができます。 <br />
この記法はスタイルシートでも利用できます。

```css:style.css
div {
    background-image: url(@scenes/quiz/img/test.png);
}
```
