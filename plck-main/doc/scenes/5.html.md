
# htmlシーン

HTMLを記述できる自由度の高いシーンです。


## `config.yaml`

`config.*.yaml` で指定できるプロパティーは以下の通りです。


| プロパティー名 | 概要                                                                                                                                                                                     | 値 |
|------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----|
| title | シーンのタイトルを指定します。<br />指定された文字列は、ヘッダーやメニューに表示されます。                                                                                                                                       | `文字列(HTMLも可能)` |
| complete_on_scroll_end | ページの一番下までスクロールされたら講座を修了状態に変更するかを設定します。 `off` に設定した場合は、シーンは修了状態になりません。<br /> 修了状態にしたい場合は、plck.config.yaml の　[`complete_flg_scene`](../unit.md#complete_flag_scene) でシーンの番号を指定する必要があります。 | `on` or `off` |
| text | HTML内で表示する文字列を定義できます。                                                                                                                                                                  | `オブジェクト` |

### テキスト指定方法

各言毎に翻訳された文字列を`html.vue`で表示させるために、`config.*.yaml` の `text` で任意の文字列を管理することができます。

#### `text` プロパティーの記述方法

`text` プロパティーは`[プロパティー名]:[値]` の形で記述することができます。

**config.ja.yaml**
```yaml
text:
    hello: こんにちは # helloというプロパティー名を指定すると、「こんにちは」という文字列が表示される
```

たとえば、別の言語で英語が指定されている場合は、同じように`hello`を定義し、英語で文字列を指定します。

**config.en.yaml**
```yaml
text:
    hello: Hello. # 言語が英語（en）の時、helloというプロパティー名を指定すると、「Hello.」という文字列が表示される
```

プロパティー名は任意の文字列を指定することができます。

プロパティーはネストしたオブジェクトや配列も解釈可能です。

**config.*.yaml**
```yaml
text:
    obj1:
        text: こんにちは ## obj1.text 
        text2: テキスト2 ## obj2.text 
    list:
        - 配列で指定1 ## list[0]
        - 配列で指定2 ## list[1]
        - 配列で指定3 ## list[2]
```

#### `text` で指定した文字列を `html.vue` で使用する

`html.vue` で文字列を指定する場合は以下のように記述します。

**config.*.yaml**
```yaml
text:
    hello: こんにちは
```

**html.vue**
```html
<template>
    <!-- <p>こんにちは</p>に変換される -->
    <p>{{text.hello}}</p>
</template>
```

`{{text.[プロパティー名]}}` で `text` で指定したプロパティーにアクセスできます。

ネストしたプロパティーや配列にアクセスする場合も以下のような JavaScriptの記法と同様の形で指定できます。

**config.*.yaml**
```yaml
text:
    obj1:
        text: こんにちは ## obj1.text 
        text2: テキスト2 ## obj2.text 
    list:
        - 配列で指定1 ## list[0]
        - 配列で指定2 ## list[1]
        - 配列で指定3 ## list[2]
```

**html.vue**

```html
<template>
    <!-- <p>こんにちは</p>に変換される -->
    <p>{{text.obj1.text}}</p>
    <!-- <p>テキスト2</p>に変換される -->
    <p>{{text.obj1.text2}}</p>
    <ul>
        <!-- <li>配列で指定1</li>に変換される -->
        <li>{{test.list[0]}}</li>
        <!-- <li>配列で指定1</li>に変換される -->
        <li>{{test.list[1]}}</li>
        <!-- <li>配列で指定1</li>に変換される -->
        <li>{{test.list[2]}}</li>
    </ul>
</template>
```


### `html.vue`の記述方法


#### `<template>` タグ

`html.vue` は、純粋な `vue3` の **単一ファイルコンポーネント** のテンプレートとして機能します。
そのため、 `vue3` で定義されているディレクティブなどを活用することも可能です。
`vue`の単一ファイルコンポーネントでは、`<template>` タグ内にHTML部分の記述を行います。

たとえば、 `v-for` を用いて、 `text` に指定した配列をループ処理して記述を簡素化することも可能です。

**html.vue**
```html
<template>
    <ul>
        <!-- text.listの配列をループ処理してオブジェクトを表示する -->
        <li v-for=(item in text.list) :key="item">{{item}}</li>
    </ul>
</template>
```

**注意:** `html.vue`では、単一ファイルコンポーネントで指定可能な `<script>` タグを用いたコンポーネントのカスタマイズは制限されています。


#### `<style>` タグ

`html.vue` に `<style>` タグを用いてスタイルシートを定義することができます。

通常だとこちらに記述したstyleは講座全体に影響する形となりますが、`scoped` 属性を用いることで、その範囲を`html.vue` に記述されているものに絞り込むことができます。

```html
<style>
    /* 全てのpタグに影響する */
    p {
        color: red;
    }
</style>

<style scoped>
    /* html.vue内のpタグのみに影響する */
    p {
        color: red;
    }
</style>
```

スタイルシートを共通化させたい場合やそうでない場合など状況に応じて適宜調整することができます。



## `customHook.js` で指定できるイベント

| イベントメソッド名 | 概要 | 引数                                                   |
|------|----|------------------------------------------------------|
| `onMount` | ページがマウントされた時に実行されます | `toCompleteCurrentScene`,`toNotCompleteCurrentScene` |
| `onUnMount` | ページが破棄され、別のページに移動する時に実行されます |                                                      |


### `onMount(toCompleteCurrentScene, toNotCompleteCurrentScene)`

`onMount`メソッドを使って講座の独自スクリプトを定義することができます。<br />
引数には `toCompleteCurrentScene` と `toNotCompleteCurrentScene` の二つのメソッドが渡されます。<br />
これらを用いて、その講座の状態に併せて完了状態を変更するといった処理を記述することができます。


**customHook.js**
```js
export const onMount = (toCompeteCurrentScene, toNotCompeteCurrentScene) => {

    // 何かしらの処理を実装...
    someScript((result) => {
        if (result === true) { 
            // 何かしらの処理を行ったら完了状態にする
            toCompleteCurrentScene()
        } else {
            // 失敗したら未完了のままにする
            toNotCompleteCurrentScene()
        }
    })


}

export const onUnmount = () => {

}


```
