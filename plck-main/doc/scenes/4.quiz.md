# quizシーン

クイズ形式の講座を表示することができるシーンです。


## `config.yaml` 

`config.*.yaml` で指定できるプロパティーは以下の通りです。


| プロパティー名                        | 概要                                                      | 値                              |
|--------------------------------|---------------------------------------------------------|--------------------------------|
| title                          | シーンのタイトルを指定します。<br />指定された文字列は、ヘッダーやメニューに表示されます。        | `文字列(HTMLも可能)`  |
| question_type                  | クイズの種類を指定します。 `select` だと選択クイズ、 `text` の場合は穴埋めクイズとなります。 | `select` or  `text` |
| domark                         | 採点を行うかを指定します。                                           | `on` or `off` |
| display_pass                   | 結果画面で合格・不合格の表示を行うかを指定します。                               | `on` or `off` |
| display_score                  | 結果画面で採点結果の点数の表示を行うかを指定します。                              | `on` or `off` |
| retry                          | 不合格（`passing_mark`よりも点数が低い）場合に、クイズの再挑戦ができるかどうかを指定します。   | `on` or `off`  |
| retry_btn_name                 | 再挑戦ボタンのラベルを指定します                                        | `文字列(HTMLも可能)` |
| block_next_when_retry          | 再挑戦となったときに次のシーンへの移動を制限するかどうかを指定します。                     | `on` or `off`  |
| passing_mark                   | 採点時の合格点数を指定します。 点数の上限は `100` です。                        | `数値(>=100)`                    |
| btn_answer_name                | 回答ボタンのラベルを指定します。                                        | `文字列(HTMLも可能)`                 |
| btn_next_name                  | 次の問題へ移動するボタンのラベルを指定します。                                 | `文字列`                          |
| btn_prev_name                  | 前の問題へ移動するボタンのラベルを指定します。                                 | `文字列`                          |
| btn_next_scene_name            | 次のシーンへへ移動するボタンのラベルを指定します。                               | `文字列`                          |
| btn_next_unit_name             | 次のユニットへ移動するボタンのラベルを指定します。                               | `文字列`                          |
| btn_end_unit_name              | ユニットを終了するボタンのラベルを指定します。                                 | `文字列`                          |
| maru_text                      | 正解したときの〇マークの文字列を指定できます。htmlも利用可能なので、画像を指定することもできます。     | `文字列(HTMLも可能)`                 |
| batsu_text                     | 不正解のときの×マークの文字列を指定できます。htmlも利用可能なので、画像を指定することもできます。     | `文字列(HTMLも可能)`                 |
| passed_text                    | 合格時の文言のラベルを指定できます。                                      | `文字列(HTMLも可能)`                 |
| failure_text                   | 不合格時の文言のラベルを指定できます。                                     | `文字列(HTMLも可能)`                 |
| table_result_text              | 結果画面のテーブルに表示される結果タイトルのラベルを指定できます                        | `文字列(HTMLも可能)`                 |
| table_answer_text              | 結果画面のテーブルに表示される回答タイトルのラベルを指定できます                        | `文字列(HTMLも可能)`                 |
| teable_maru_text               | 結果画面のテーブルで表示される正解したときのラベルを指定できます。                       | `文字列(HTMLも可能)`                 |
| teable_batsu_text              | 結果画面のテーブルで表示される不正解したときのラベルを指定できます。                      | `文字列(HTMLも可能)`                 |
| confirm_table_description_text | 確認画面でのテーブルクリック時の文言を指定できます。                              | `文字列(HTMLも可能)`                 |
| mark_table_description_text    | 結果画面でのテーブルクリック時の文言を指定できます。                              | `文字列(HTMLも可能)`                 |
| guide_pretext                  | 解説のタイトルのラベルを指定できます。                                     | `文字列(HTMLも可能)`                 |
| questions                      | 問題を指定できます。詳細は[こちら](#questions)をご参照ください                  | `配列` [questions](#questions)[] |

### `questions` オブジェクト

`questions` は、各問題を定義するオブジェクトです。

| プロパティー名       | 概要                                                                                  | 値                       |
|---------------|-------------------------------------------------------------------------------------|-------------------------|
| quiz_type     | クイズの種類を指定します。 `select` と `text` から選択できます。                                           | `select` or `text`      |
| alternatively | 回答が複数回答か、単一回答かを選択する。 `single` を指定すると単一回答・ `multiple`だと複数回答となります。                    | `multiple` or `single`  |
| title         | クイズのタイトルを指定します。                                                                     | `文字列`                   |
| quiz_text     | クイズの本文を指定します。                                                                       | `文字列`                   |
| choices       | `quiz_type` が `select` の場合に指定できます。クイズの選択肢を配列で指定できます。                                | `配列: string[]`          |
| text          | `quiz_type` が `text` の場合に指定できます。クイズの穴埋め問題を文字列で指定できます。<br />指定方法は以下を参照ください           | `文字列`                   |
| correct       | 回答を指定します。`alternatively` が `single` の場合は答えとなる文字列を、`multiple` の場合は答えとなる文字列の配列を指定します。 | `文字列` or `配列: string[]` |
| explanation   | 回答の解説文を指定します。                                                                       | `文字列(HTMLも可能)`          |


### `questions` の指定方法

`questions` で問題を指定します。<br />
問題は 選択式と穴埋め問題式と2つを指定することができます。

**選択式の場合**
```yaml
- quiz_type: select
  alternatively: single
  title: 第1問
  quiz_text: 問題文を入力してください
  choices:
    - 選択肢1
    - 選択肢2
    - 選択肢3
  correct: 1
  explanation: 解説文を入力してください
```

選択式の場合は、`quiz_type` に `select` を指定します。<br />
合わせて `choices` で選択肢を指定する必要があります。

`alternatively` が `single` の場合は選択フォームがラジオボタンとなり、 `multiple` の場合は、チェックボックスになります。

**選択式の場合の、`correct` の指定について**

選択式の場合、 `correct` には正解となる選択肢の番号を入力します。
例えば、1番目の選択肢が正解の場合は、

```yaml
- quiz_type: select
  alternatively: single
  ...
  choices:
    - 選択肢1
    - 選択肢2
    - 選択肢3
  correct: 1
...
```
という形で、 `correct` に　`1` と入力します。<br />

`multiple` の場合はこれを配列にします。<br />
1番目と3番目の選択肢を正解とする場合は、

```yaml
- quiz_type: select
  alternatively: multiple
  ...
  choices:
    - 選択肢1
    - 選択肢2
    - 選択肢3
  correct:
    - 1
    - 3
...
```
上記のように　配列で2つの選択肢の番号を配列で入力します。



**穴埋め問題式の場合**
```yaml
- quiz_type: text
  alternatively: single
  title: 第4問
  quiz_text: 問題文を入力してください。その3
  text: |
    %input% を入力する<br />
    こんにちは
  correct: 文字列
  explanation: 解説文を入力してください。その3
```

`quiz_type` を `text` に指定すると、穴埋め問題式になります。<br />
`text` で問題文を指定できます。

`text`の問題文の中に `%input%` と記述すると、実際に画面上で表示される際にインプットテキスト要素に代わります。<br />

```yaml
%input% を入力する<br />
こんにちは
```

`alternatively` が `multiple` の場合は、`%input[0]%` のように 後ろに `[{番号}]` と記述することで複数の問題文を定義することができます。<br />
`{番号}`の部分は 0から始まる数値を指定することができます。<br />
番号が `correct` で指定されている解答の順番に対応します。

```yaml
- quiz_type: text
  alternatively: multiple
  title: 第4問
  quiz_text: 問題文を入力してください。その3
  text: |
    %input[0]% を入力する<br />
    %input[1]% も入力する<br />
    こんにちは<br />
    よろしくお願いします。
  correct:
    - 文字列1 # %input[0]% の解答内容
    - 文字列2 # %input[1]% の解答内容
```

|-------------------|-------------------------------------|----|
| `quizInitialized` | クイズが初期化された時（初めて表示される時）に実行されるイベントです。 | なし |
