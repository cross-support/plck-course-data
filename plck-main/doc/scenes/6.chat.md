
# chatシーン

チャット形式のHTMLを記述できる自由度の高いシーンです。


## `config.yaml`

`config.*.yaml` で指定できるプロパティーは以下の通りです。


| プロパティー名 | 概要                                                                                                                                                                                      | 値 |
|------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----|
| title | シーンのタイトルを指定します。<br />指定された文字列は、ヘッダーやメニューに表示されます。                                                                                                                                        | `文字列(HTMLも可能)` |
| complete_on_scroll_end | ページの一番下までスクロールされたら講座を修了状態に変更するかを設定します。　 `off` に設定した場合は、シーンは修了状態になりません。<br /> 修了状態にしたい場合は、plck.config.yaml の　[`complete_flg_scene`](../unit.md#complete_flag_scene) でシーンの番号を指定する必要があります。 | `on` or `off` |
| text | HTML内で表示する文字列を定義できます。                                                                                                                                                                   | `オブジェクト` |
| chat | HTML内で表示する会話を定義できます。                                                                                                                                                                    | `オブジェクト` |


### `chat`シーンについて

chatシーンは `html`シーンを拡張したものであるため、 `text`プロパティーの定義方法や`html.vue` の記述方法は `html` シーンと同じです。<br />
`html` シーンについては[こちら](./5.html.md)をご参照ください。


### 会話の作成

#### `chat`プロパティーの設定

`chat`プロパティーでは、会話をスレッドごとに定義することで、独立した会話を複数定義することができます。

```yaml
chat:
    thread1: 
        ...# thread1の会話内容を記述
    thread2:
        ...# thread2の会話内容を記述
```

スレッド名は任意で定義できます。<br />

各会話のセリフ部分は以下のオブジェクトを配列で指定する必要があります。

**`serif`オブジェクト**

| プロパティー名 | 概要 | 値 |
|------|----|----|
| image | 会話のアバター画像を指定できます。 | `文字列` |
| textHtml | 会話の内容を指定できます。HTMLも利用することができます。 | `文字列(HTMLも可能)` |
| right | 会話を右揃えにするかを指定します。 | `on` or `off` |


```yaml
chat:
    thread1:
        - image: '@scenes/chat/chatImg/avatar01.png'
          textHtml: 会話の文字列を入力します
        - image: '@scenes/chat/chatImg/avatar01.png'
          textHtml: 会話の文字列を入力します。<strong>この会話は右揃えで表示されます</strong>
          right: on
        ...

```

### `html.vue`で会話を表示する

`html.vue` で会話を表示する場合には、PLCKの独自タグである `<talk-context>` タグを使用します。<br />
会話を`talk`属性を用いて指定します。

```html
<template>
    <!-- chat.trhead1の会話を表示する -->
    <talk-context :talk="chat.thread1" />

    <!-- chat.trhead2の会話を表示する -->
    <talk-context :talk="chat.thread2" />
</template>
```



## `customHook.js` で指定できるイベント

| イベントメソッド名 | 概要 | 引数 |
|------|----|----|
| `onMount` | ページがマウントされた時に実行されます | `toCompleteCurrentScene`,`toUnCompleteCurrentScene` |
| `onUnMount` | ページが破棄され、別のページに移動する時に実行されます | |


### `onMount(toCompleteCurrentScene, toUnCompleteCurrentScene)`

`onMount`メソッドを使って講座の独自スクリプトを定義することができます。<br />
引数には `toCompleteCurrentScene` と `toUnCompleteCurrentScene` の二つのメソッドが渡されます。<br />
これらを用いて、その講座の状態に併せて完了状態を変更するといった処理を記述することができます。


**customHook.js**
```js
export const onMount = (toCompeteCurrentScene, toUnCompeteCurrentScene) => {

    // 何かしらの処理を実装...
    someScript((result) => {
        if (result === true) { 
            // 何かしらの処理を行ったら完了状態にする
            toCompleteCurrentScene()
        } else {
            // 失敗したら未完了のままにする
            toUnCompleteCurrentScene()
        }
    })


}

export const onUnmount = () => {

}


```
