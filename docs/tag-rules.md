# docs/tag-rules.md — 言語タグ・カテゴリタグ運用ルール

LMS / 管理画面 / サイトでの講座分類のためのタグ運用ルール。
**翻訳講座が増えるたびに、タグ付け漏れを起こさないこと**が本ドキュメントの目的。

---

## 1. タグ運用の目的

- 受講者が **言語別** に講座を絞り込めるようにする
- 受講者が **カテゴリ別**（ロジスティクス、ビジネスマナー 等）に講座を絞り込めるようにする
- 管理者が講座セットを横断して品質確認できるようにする
- 多言語展開時の運用ミス（タグ付け漏れ）を防ぐ

---

## 2. 言語タグ

### 2-1. 言語タグ一覧

| 言語タグ | prefix | 括弧語表記 |
|---|---|---|
| 日本語 | `ja-` / `privacy-` / `security-` | （日本語）または省略 |
| 英語 | `en-` | （English） |
| ポルトガル語 | `pt-` | （Português） |
| ベトナム語 | `vi-` | （ベトナム語） |
| 中国語 | `zh-` | （中国語） |
| 韓国語 | `ko-`（今後） | （한국어） |
| タガログ語 | `tl-`（今後） | （Tagalog） |

### 2-2. 必ず確認すべきこと

新しい翻訳講座が追加されたら、必ず:

1. **言語タグが付いているか**を管理画面で確認
2. **括弧語表記**が講座名カード・サムネイル・LMS 表示・カテゴリページのすべてで一貫しているか確認
3. 既存講座（英語・ポルトガル語）と表示仕様が揃っているか目視確認

---

## 3. カテゴリタグ

### 3-1. カテゴリタグ一覧

| カテゴリタグ | 対象講座（例） |
|---|---|
| ロジスティクス | ベトナム語/中国語 ロジスティクス入門 Ⅰ〜Ⅳ など |
| ビジネスマナー | ビジネスマナー基礎、応用 など |
| 安全教育 | 労働安全、フォークリフト操作 など |
| 品質意識 | QC 入門、5S 活動 など |
| 電話応対 | 電話応対の基本、クレーム対応 など |
| ビジネス文書 | メール作成、報告書 など |
| 報告書・議事録 | 議事録の取り方、要約スキル など |
| コンプライアンス | 個人情報の取扱、情報セキュリティ など |

> 既存講座にすでに付与されているタグに合わせる。新規カテゴリは PM 承認の上で追加。

### 3-2. 必ず確認すべきこと

ロジスティクス系の翻訳講座（vi-logistics0X / zh-logistics0X）には、必ず:

- **ロジスティクスのカテゴリタグ**が付与されているか
- 言語タグ + ロジスティクスタグの **両方** が付いているか

を確認すること。**言語タグだけ付けてカテゴリタグを忘れる**事故が起きやすい。

---

## 4. ClaudeCode への確認ルール（テンプレ）

翻訳講座が追加されたとき、ClaudeCode に次のように依頼する:

```
docs/tag-rules.md を読んでください。

今回追加された講座は「{講座名}（{言語}）」です。以下を確認してください:

1. この講座に対応する言語タグ（例: 中国語）が付いているか
2. 講座カテゴリタグ（例: ロジスティクス）が付いているか
3. 英語・ポルトガル語で成立している表示仕様（括弧語・表示名）が、
   この言語にも反映されているか
4. 既存講座と並べて違和感（タグ漏れ・表記揺れ）がないか
5. 違反項目があれば、具体的な修正案を提示してください
```

---

## 5. タグ整合性チェックリスト

新言語講座 / 新カテゴリ講座の追加後に通す:

- [ ] 言語タグが付与されている
- [ ] カテゴリタグが付与されている
- [ ] 言語タグとカテゴリタグの両方が同時に絞り込めるか管理画面で確認
- [ ] 講座名カードの括弧語表記が他言語と一貫している
- [ ] LMS 受講者画面で言語別フィルタが動作する
- [ ] LMS 受講者画面でカテゴリ別フィルタが動作する
- [ ] 管理画面の講座一覧で並び替え時に違和感がない
- [ ] サムネイル・表示名・括弧語の 3 か所が一致

---

## 6. タグ付与の責任分担

| ロール | 責任範囲 |
|---|---|
| 講座制作担当 | 講座作成時に言語タグ・カテゴリタグを付与 |
| 翻訳担当 | 翻訳追加時に言語タグの付与を確認 |
| LMS 管理者 | LMS への搭載時にタグ整合性を確認 |
| ClaudeCode | 上記すべての確認を機械的に実行・報告 |

---

## 7. 既存講座のタグ運用（参考）

| 講座 | 言語タグ | カテゴリタグ | 確立済み運用 |
|---|---|---|---|
| 個人情報の取扱 | 日本語 | コンプライアンス | ◎ |
| 情報セキュリティ | 日本語 | コンプライアンス | ◎ |
| ベトナム語_よくわかる！ロジスティクス Ⅰ〜Ⅳ | ベトナム語 | ロジスティクス | ◎ |
| 中国語_よくわかる！ロジスティクス Ⅰ〜Ⅳ | 中国語 | ロジスティクス | ◎ |

新規追加時は、上記の運用に合わせる。

---

## 8. 表示仕様の整合性

タグ付与だけでなく、**表示仕様**も既存講座と揃える。
詳細は [translation-rules.md §8](translation-rules.md) を参照。

主な確認ポイント:

- 講座名の表示順（言語名カッコ書きが先か後か）
- フォント・配色（ヘッダー色 #717d85、下ボーダー緑 #98c9b2 等）
- ハンバーガーメニュー非表示
- 完了アイコン★非表示
