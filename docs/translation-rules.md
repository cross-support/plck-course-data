# docs/translation-rules.md — 翻訳追加時の言語別ルール

新しい言語の講座を追加するときに従うべきルール。
**英語・ポルトガル語が今後追加される前提**で、ベトナム語・中国語と同じ品質基準で扱うこと。

> 🚨 **絶対ルール**: フォント禁則の根拠は [LMS_BUILD_RULES.md §4](../LMS_BUILD_RULES.md) を参照。

---

## 1. 言語別ビルドルール表

| 言語 | prefix | フォント置換 (Meiryo→Arial) | PPTX→PNG 再レンダリング | 推奨 PNG 入手手段 | よくある崩れ | 主な対策 |
|---|---|---|---|---|---|---|
| **日本語** | `ja-` / `privacy-` / `security-` | 許可 | 許可 | PPTX → 200DPI PNG 変換 | 行間・カーニング差 | `pdftoppm -r 200` 固定 |
| **英語** | `en-` | 許可（Arial に英字字形が完備） | 許可 | PPTX → 200DPI PNG | カーニング差・字体差 | フォント明示指定 |
| **ポルトガル語** | `pt-` | 許可（ラテン拡張・チルダ `ã ç` も Arial 描画可） | 許可 | PPTX → 200DPI PNG | チルダ・アクセント分離 | 検証スライドで目視確認 |
| **ベトナム語** | `vi-` | **許可**（Arial が声調記号を含む） | 許可 | `tools/scripts/pptx_to_png.sh` | 声調記号の分離（`á → a ´`）| Arial 置換込みで使用 |
| **中国語** | `zh-` | **🚫 禁止** | **🚫 原則禁止** | デザイン担当出力 PNG をそのまま使用 | 明朝化・字形崩壊 | 元 PNG とハッシュ一致必須 |
| **韓国語** | `ko-`（今後） | **🚫 禁止**（Hangul 字形なし） | 検証前は禁止 | デザイン出力直接使用 | 中国語と同類 | preflight に ko-\* 検査追加 |
| **タガログ語** | `tl-`（今後） | 要検証（ラテン拡張） | 検証後に許可 | PPTX → PNG（要 QA） | ñ・アクセントの分離 | 実装時に QA 必須 |
| **その他** | * | 禁止（個別検証承認後に追加） | 禁止 | デザイン出力直接使用 | 未知 | 表に追記 |

## 2. ⚠️ 中国語（zh-\*）特例ルール

### 2-1. なぜ Arial 置換が禁止か

- Arial フォントは **CJK（漢字）字形を持たない**
- OS 描画時に Arial → 明朝系（Songti、SimSun 等）にフォールバック
- 元 PowerPoint のゴシック太字が **明朝細字** に化ける
- 「文字化けではない」が、デザインとして別物の PNG になる
- 受講者には**フォント崩れスライド**として表示される

### 2-2. 中国語講座で守るべきこと

1. `tools/scripts/pptx_to_png.sh` / `build_unit.sh` を **使わない**
2. デザイン担当が PowerPoint / 専用ツールで出力した PNG をそのまま使う
3. `plck-main/contents/scenes/slide/zh-*/slide/*.png` の SHA-256 を元 PNG と完全一致に保つ
4. preflight が SHA 不一致を検出したら、必ず元 PNG で上書きし直してから再ビルド
5. PowerPoint で再レンダリングが必要な場合は、フォントを **Microsoft YaHei / Noto Sans SC / PingFang SC** で**明示指定**してから出力

### 2-3. preflight が中国語講座で追加検査すること

- `plck-main/contents/scenes/slide/zh-*/slide/` に `.pptx` / `.pdf` が混入していないか
- ハッシュ不一致のときに「中国語: Meiryo→Arial 置換の副作用が疑われる」エラーメッセージを出す

## 3. ベトナム語（vi-\*）特例ルール

### 3-1. なぜ Arial 置換が許可か

- Arial にラテン拡張（声調記号）が完備
- Meiryo にも一部声調はあるが、合成済みグリフ（precomposed）と分解形（decomposed）の差で**記号分離**が起きやすい
- Arial 置換で安定描画される

### 3-2. ベトナム語で確認すべきこと

- á à ả ã ạ（5 種の a）
- ă â ê ô ơ ư đ
- 上記が **文字と分離せず**、1 字として描画されているか目視確認

### 3-3. ベトナム語ビルドコマンド

```bash
./tools/scripts/pptx_to_png.sh \
    "/path/to/UNIT1.pptx" \
    "/path/to/講座フォルダ/PNG/UNIT1"

# あるいは
./tools/scripts/build_unit.sh \
    vi-logistics01-unit01 \
    "/path/to/UNIT1.pptx" \
    "/path/to/講座フォルダ/LMS搭載用ZIP"
```

## 4. 英語（en-\*）追加時のルール（今後の予定）

### 4-1. 言語タグ

- 管理画面で「英語」タグを登録
- 講座カード・サムネイル・LMS 表示に「（English）」括弧語を付与（既存ベトナム語・中国語と同じ運用）

### 4-2. ビルド

- `tools/scripts/pptx_to_png.sh` を使用可（Arial で英字は問題なし）
- preflight の `PAIRS` に追記:

```bash
"en-logistics01|英語_よくわかる！ロジスティクス入門Ⅰ"
```

### 4-3. 表示確認

- 行折り返し位置（CJK と異なるため、レイアウトが大きく変わる場合がある）
- 数字 / 単位 / 句読点（コンマ・ピリオド）

---

## 5. ポルトガル語（pt-\*）追加時のルール（今後の予定）

### 5-1. 言語タグ

- 管理画面で「ポルトガル語」タグを登録
- 括弧語: 「（Português）」（既存運用に合わせる）

### 5-2. ビルド

- Arial にチルダ（`ã ç`）・アクセント（`é á í ó ú`）字形あり → 置換可
- `tools/scripts/pptx_to_png.sh` を使用してよい

### 5-3. 表示確認

- ç ã õ á é í ó ú の字形が分離していないか
- 行折り返し（英語と同様の問題）

## 6. 言語追加時の必須チェックリスト

新言語を追加するときは **このリストを通す**:

- [ ] 言語タグが管理画面に登録されているか
- [ ] 括弧語表記（例: `（中国語）` `（English）` `（Português）`）が登録されているか
- [ ] 既存（英語・ポルトガル語で成立している）表示仕様が新言語にも反映されるか
- [ ] ロジスティクス系の場合、**ロジスティクスのカテゴリタグ** が付いているか
- [ ] preflight.sh / build_all.sh の `PAIRS` に追記したか
- [ ] PNG 入手手段が「PPTX→自動変換可」「デザイン出力必須」のどちらか確定したか
- [ ] フォント置換の可否を [LMS_BUILD_RULES.md §4](../LMS_BUILD_RULES.md) 表に追記したか
- [ ] 代表 1 UNIT で目視確認（[LMS_BUILD_RULES.md §4-A](../LMS_BUILD_RULES.md)）を通したか
- [ ] 言語別の頻出崩れパターン（声調・チルダ・CJK 化け 等）を `troubleshooting.md` に追記したか

## 7. ClaudeCode への確認指示テンプレ

新言語講座を追加するときに ClaudeCode に投げる指示の例:

```
今回追加する言語は「英語」です（prefix: en-）。
docs/translation-rules.md と docs/tag-rules.md を読んで、以下を実施してください:

1. 言語タグ「英語」と括弧語「（English）」が、既存ベトナム語・中国語と同じ運用で
   付与されるかを確認する
2. ロジスティクス系の場合は「ロジスティクス」カテゴリタグが付いているか確認する
3. preflight.sh / build_all.sh の PAIRS に英語講座を追記する
4. 代表 1 UNIT で目視確認（LMS_BUILD_RULES.md §4-A）を通す
5. 違反項目があれば修正案を提示する
```

## 8. 言語別 表示仕様の整合性確認

英語・ポルトガル語が **先に確立した表示仕様** を、ベトナム語・中国語にも反映する:

| 表示要素 | 英語 | ポルトガル語 | ベトナム語 | 中国語 |
|---|---|---|---|---|
| 講座名カード括弧語 | （English）| （Português） | （ベトナム語） | （中国語） |
| ヘッダー色 | グレー #717d85 | グレー #717d85 | グレー #717d85 | グレー #717d85 |
| ヘッダー下ボーダー | 緑 #98c9b2 4px | 緑 #98c9b2 4px | 緑 #98c9b2 4px | 緑 #98c9b2 4px |
| タイトル行 | 1 行 | 1 行 | 1 行 | 1 行 |
| ハンバーガー | 非表示 | 非表示 | 非表示 | 非表示 |
| 完了アイコン★ | 非表示 | 非表示 | 非表示 | 非表示 |

新言語追加時は **この表をすべて満たすこと**。違反があれば[LMS_BUILD_RULES.md §3](../LMS_BUILD_RULES.md) の CSS 仕様で揃える。
